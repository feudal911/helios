Solução e Arquitetura

Arquitetura do Sistema

Padrão MVC (Model-View-Controller)

O HELIOS foi desenvolvido seguindo o padrão arquitetural MVC para garantir separação de responsabilidades e facilidade de manutenção:

  • Model: Gerencia a lógica de negócio e interação com o banco de dados
  • View: Templates HTML/Jinja2 responsáveis pela interface do usuário
  • Controller: Rotas Flask que recebem requisições e interagem com Models e Views
Tecnologias Utilizadas
  • Backend: Python Flask
  • Banco de Dados: SQLite (SQLAlchemy ORM)
  • Autenticação: Flask-Login com sessões
  • Segurança: Bcrypt para hash de senhas
  • Frontend: Bootstrap 5 + Jinja2
  • API: RESTful para telemetria
Funcionalidades Principais
  • CRUD completo de Parques Solares
  • CRUD completo de Inversores/Ativos
  • CRUD completo de Regras de Alerta
  • Upload de dados históricos (CSV)
  • Download de relatórios (PDF/CSV)
  • API RESTful para telemetria
  • Sistema de alertas preditivos
Estrutura de Dados

O sistema utiliza as seguintes entidades principais:

  • Usuarios: Técnicos e administradores do sistema
  • Parques: Fazendas solares cadastradas
  • Inversores: Equipamentos de inversão de energia
  • Regras: Regras lógicas para geração de alertas
  • Medicoes_Telemetria: Dados de medição em tempo real
  • Alertas: Alertas gerados pelas regras
